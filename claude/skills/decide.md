# Decide

When Yulong is stuck on a decision, walk him through this framework.

**Mantra: gut. satisfice. commit.**

## How to use

1. Ask him to state the decision in one sentence.
2. Run the 6 steps. Stop when an answer lands.
3. If he expands the question, redirect — this closes loops, doesn't open them.
4. After he commits: *don't relitigate for 24h.*

## The 6 Steps

1. **Gut** — first instinct, 10 seconds, no reasoning
2. **Energy** — which option energises? which drains?
3. **Satisfice** — is one option "good enough"? (he over-maximises)
4. **1-year test** — will you care in a year? (usually no)
5. **Worst case** — what's the actual worst? (usually: fine)
6. **Commit** — pick, tell Ross or me, don't re-evaluate for 24h

## Action heuristics (if still stuck)

- **Smallest step** — what's the tiniest micro-decision in the next few minutes?
- **5-second rule** — for low-stakes, count down 5,4,3,2,1, act
- **Progress > perfection** — wrong decision made fast beats right one made late
- **Waiting is a decision too** — inaction has its own cost (stress, spinning time)
- **30% / 80% rule** — act 30% sooner than comfortable, or when plan is 80% good

## Red Flags to Catch

- "But what about X?" for the 5th+ time → looking for permission to not decide
- Optimising a sub-decision → avoiding the binary above it
- Circling back after "deciding" → that option is pulling him
- Deciding for >2 days → the indecision IS the problem

## Escalation

- Raw fear (not overthinking) → reassurance + gradual steps
- Decision involves Ross → ask him first
- Bedtime or exhaustion → defer, don't decide depleted
