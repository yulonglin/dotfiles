# Context

Add Mouseless (net.sonuscape.mouseless) config to dotfiles so it survives machine migrations and stays in sync across machines.

**Config location:** `~/Library/Containers/net.sonuscape.mouseless/Data/.mouseless/configs/config.yaml`
- macOS sandboxed app — no `--config` flag, fixed path in container
- App writes config atomically via `write-to-temp + rename()` on UI save → destroys symlinks
- Machine-specific fields to strip: `app_version` (dirty on every app update), `keyboard_layout` (200+ line auto-generated vk_to_char table; Mouseless regenerates if absent)

**Why copy, not symlink:** `rename()` replaces the symlink inode — same issue as htop in this repo.

**Deferred:** Making core components (shell/tmux/git) flag-gatable is tracked separately in
`plans/deploy-core-component-flags.md` (full absolute path below).

# Plan

## 1. Initial snapshot: copy config into dotfiles (stripped)

Run once from the dotfiles root:
```bash
mkdir -p config/mouseless
python3 -c "
import yaml
import os
src = os.path.expanduser('~/Library/Containers/net.sonuscape.mouseless/Data/.mouseless/configs/config.yaml')
dst = 'config/mouseless/config.yaml'
with open(src) as f:
    cfg = yaml.safe_load(f)
cfg.pop('app_version', None)
cfg.pop('keyboard_layout', None)
with open(dst, 'w') as f:
    yaml.dump(cfg, f, allow_unicode=True, sort_keys=False, default_flow_style=False)
"
```

Stores only portable settings (behavior, grid, keymaps, style). Verified: produces 246 lines (down from 462), all 11 portable keys preserved.

## 2. Add `sync-mouseless` to `config/aliases.sh`

Follows the `sync-secrets` pattern — encapsulates the strip logic for the occasional "save UI changes back to dotfiles" workflow:
```bash
# Sync Mouseless UI config changes back to dotfiles (macOS only)
if [[ "$(uname)" == "Darwin" ]]; then
    sync-mouseless() {
        local src="$HOME/Library/Containers/net.sonuscape.mouseless/Data/.mouseless/configs/config.yaml"
        local dst="$DOT_DIR/config/mouseless/config.yaml"
        python3 -c "
import yaml, os
src = os.path.expanduser('$src')
with open(src) as f: cfg = yaml.safe_load(f)
cfg.pop('keyboard_layout', None); cfg.pop('app_version', None)
with open('$dst', 'w') as f: yaml.dump(cfg, f, allow_unicode=True, sort_keys=False, default_flow_style=False)
"
        echo "Synced Mouseless config → $dst (stripped keyboard_layout, app_version)"
    }
fi
```

## 3. Add deployment block to `deploy.sh`

Add after the Serena block (~line 572):

```bash
# ─── Mouseless ────────────────────────────────────────────────────────────────

if [[ "$DEPLOY_MOUSELESS" == "true" ]]; then
    if is_macos; then
        log_info "Deploying Mouseless configuration..."

        MOUSELESS_DIR="$HOME/Library/Containers/net.sonuscape.mouseless/Data/.mouseless/configs"
        MOUSELESS_CONFIG="$MOUSELESS_DIR/config.yaml"
        MOUSELESS_DOTFILES="$DOT_DIR/config/mouseless/config.yaml"

        if [[ ! -f "$MOUSELESS_DOTFILES" ]]; then
            log_warning "Mouseless config not found at $MOUSELESS_DOTFILES"
        elif [[ ! -d "$MOUSELESS_DIR" ]]; then
            log_warning "Mouseless container not found — open Mouseless.app once to initialize it, then re-run ./deploy.sh --mouseless"
        else
            if [[ -f "$MOUSELESS_CONFIG" && ! -L "$MOUSELESS_CONFIG" ]]; then
                backup_file "$MOUSELESS_CONFIG"
            fi
            cp "$MOUSELESS_DOTFILES" "$MOUSELESS_CONFIG" && \
                log_success "Deployed Mouseless configuration" && \
                log_info "  keyboard_layout and app_version regenerated by Mouseless on first launch" && \
                log_info "  To sync UI changes back: sync-mouseless"
        fi
    else
        log_info "Mouseless is macOS-only, skipping"
    fi
fi
```

Key decisions:
- **`cp`** not `safe_symlink` — atomic rename on UI save destroys symlinks
- **No `mkdir -p "$MOUSELESS_DIR"`** — sandbox daemon creates it with correct entitlements; manually creating could break sandbox
- **Positive-first `is_macos`** — matches Ghostty convention
- **`cp ... && log_success`** — guards success message against copy failure
